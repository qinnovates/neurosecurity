Purpose & Mission

Autodidactive is a self-directed learning tool built for anyone entering the intersection of neuroscience, security, and ethics. It was built by Kevin Qi as a personal learning companion while studying neurosecurity — and is shared openly so others can use the same structured approach.

The mission: make the foundational knowledge behind brain-computer interface security accessible to newcomers, while growing alongside the creator's own education. Every concept, person, and framework in here maps back to the work being done at Qinnovate on BCI security.

How to access: Visit qinnovate.com/learn/autodidactive/ in any browser. Works on mobile, tablet, and desktop. Add to your home screen for a native app experience (PWA). No account needed. No data leaves your device.

Each category expands into sub-pills. For example, Security contains Pioneers (the people who built cybersecurity), TARA Threats (109 attack techniques), NISS Scoring (22 device risk scores), and Neurowall (neural firewall architecture).

QIF Content (Qinnovate Integration Framework)

All of Qinnovate's neurosecurity research is integrated directly into the learning experience:

  • TARA Threats — 109 attack techniques mapped across the QIF hourglass, with status, severity, and dual-use classification
  • NISS Scoring — 22 BCI devices scored across 5 neurorights dimensions (Cognitive Liberty, Mental Integrity, Mental Privacy, Psychological Continuity, Equal Access)
  • Neurowall — Neural firewall architecture with inbound/outbound pipelines, NSP post-quantum crypto, and Runemate policy engine
  • Guardrails — 8 neuroethics guardrails from published scholarship (Morse, Poldrack, Racine/Illes, Ienca, Kellmeyer, Wexler, Tennison/Moreno, Vul/Eklund)
  • Governance — Neurorights table, global regulatory landscape (7 jurisdictions), 8 policy documents, and DSM-5-TR clinical mapping
  • Brain Atlas — 37 brain regions across 7 neural bands with functions, recording methods, and clinical relevance
  • BCI Devices — 26 companies and 34 devices with funding, FDA status, channel counts, and security posture
  • QIF Hourglass — The 7-1-3 architecture model (7 neural bands, 1 interface, 3 silicon bands) with physics reasoning
Important: QIF, NISS, TARA, NSP, Neurowall, and Runemate are all proposed frameworks by Qinnovate. They have not been independently validated or adopted by any standards body. Techniques marked CONFIRMED are documented in published literature; others are theoretical projections for defensive planning.

Notes & Mind Mapping

The Notes tab is a personal thinking space:

  • Note Wall — Free-placement post-it notes with drag, color picker, and automatic timestamps
  • "Let's mind my business!" — Click this button and the app analyzes your notes locally using TF-IDF cosine similarity + knowledge graph matching to auto-generate a mind map showing hidden connections between your ideas
  • Time clustering — Notes taken in the same session are grouped by color
  • Privacy by design — All NLP analysis runs client-side in JavaScript. Nothing is transmitted to any server

Other Features

  • Daily Discovery — Seeded PRNG shows one figure per day, same for everyone
  • 10 Guided Paths — Step-by-step learning journeys (e.g., "First Principles," "BCI Security 101," "Neuroethics Foundations")
  • Knowledge Graph — 45 people, 47 nodes, 578+ edges connecting concepts across disciplines
  • Search — Full-text search across 1,198 indexed entries
  • Ethics Timeline — 73 events from ~2400 BCE to present, filterable by tradition
  • Spaced Repetition — SM-2 algorithm tracks what you've learned
  • Audio — Curated educational audio content
  • AI Tutor — Optional AI-assisted learning (requires API key, runs locally)
  • PWA Ready — Add to home screen on mobile for app-like experience

Architecture

  • Zero dependencies — Pure ES modules, no npm, no build step, no framework
  • Vanilla JS — All logic in plain JavaScript with ES module imports
  • Lazy loading — Large data files (TARA, NISS, BCI, Brain Atlas) loaded on demand via dynamic import()
  • localStorage only — All user data stays in the browser. No accounts, no servers, no tracking
  • Dual maintenance — Source in public/learn/autodidactive/, deployed copy in docs/learn/autodidactive/

Scaling with Local AI

The mind mapping feature currently uses TF-IDF cosine similarity — a lightweight NLP technique that runs instantly in the browser. For richer analysis at scale, the architecture supports:

  • WebLLM — Run full language models (Llama, Phi) in the browser via WebGPU. Same privacy guarantee, deeper semantic understanding
  • Transformers.js — Hugging Face models compiled to ONNX for browser execution. Sentence embeddings, classification, summarization
  • On-device inference — No cloud API required. Models download once and run locally. Neural data and personal notes never leave the device

The principle is the same at every scale: your data stays yours. The model comes to your device, not the other way around.
