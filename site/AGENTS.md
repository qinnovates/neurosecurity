# docs/ -- GitHub Pages Build Output + Working Documents

## Structure (Dual Nature)
- Root-level files (`index.html`, `_astro/`, `CNAME`, `sitemap-*.xml`, `rss.xml`) = **build output**. Do NOT edit directly
- `learn/` -- Learning materials (autodidactive curriculum)
- `product/` -- Product specs (demo atlas wireframes, design docs)
- `research/` -- Research documents (cross-AI reviews, validation, whitepaper drafts)
- `data/` -- Gitignored prebuild output (Parquet files, JSON, KQL tables)

## Conventions
- Never edit HTML files in `docs/` directly. Edit source in `src/` and run `npm run build`
- Build output is committed to `docs/` for GitHub Pages deployment
- `docs/data/` is regenerated by `npm run prebuild` -- treat as ephemeral
- Working documents in `learn/`, `product/`, `research/` are editable markdown
- The `CNAME` file controls the custom domain (qinnovate.com)
