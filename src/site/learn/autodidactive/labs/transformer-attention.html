Transformer

Vaswani et al., 2017 — "Attention Is All You Need"

Every word asks a question (Q), offers a label (K), and carries information (V). Attention = matching questions to labels, then reading the info.

Q

Query
"What am I looking for?"
Each token generates a Query vector. It's what this token wants to know from the rest of the sequence. Like a search query.

K

Key
"Here's my label."
Each token also generates a Key vector. Keys are compared against Queries to determine relevance. Like a file label that gets matched to search terms.

V

Value
"Here's my actual content."
The actual information payload. Once Q and K determine how much attention to pay, V is what gets read. Like the file contents behind the label.

How They Work Together

Attention(Q, K, V) = softmax(Q × KT / √dk) × V

Step 1: Multiply Q by KT — this gives a score for every pair of tokens. High score = "these two tokens are relevant to each other."

Step 2: Divide by √dk — prevents scores from getting too large (stabilizes gradients).

Step 3: Softmax — converts scores into probabilities (they sum to 1).

Step 4: Multiply by V — each token's output is now a weighted mix of all Value vectors, weighted by relevance.

Self-Attention

In self-attention, Q, K, and V all come from the same input sequence. Each word "queries" every other word to decide what's relevant.

The heatmap shows the attention matrix — each cell is the dot product between one word's Query and another word's Key, after softmax.

"it" strongly attends to "cat" because the model learned that pronouns reference earlier nouns. This is the core insight — no recurrence needed, just pattern matching across the whole sequence at once.

This replaces the sequential processing of RNNs. Instead of hoping information about "cat" survives through 5 time steps to reach "it", the transformer directly connects them in one operation.

One attention head can only focus on one type of relationship. Multiple heads let the model see grammar, meaning, position, and coreference simultaneously.

Why Multiple Heads?

The original paper uses 8 parallel attention heads. Each head has its own Q, K, V projection matrices, so each learns to focus on different linguistic relationships.

After all heads compute their attention, the results are concatenated and passed through a linear layer. This gives the model a rich, multi-perspective view of the input.

Think of it like 8 people reading the same sentence, each highlighting different things — one marks grammar, another marks who-refers-to-what, another marks semantic similarity.

Transformers have no built-in sense of word order. Positional encodings inject position information using sine and cosine waves at different frequencies.

How It Works

PE(pos, 2i) = sin(pos / 100002i/d)
PE(pos, 2i+1) = cos(pos / 100002i/d)

Each position gets a unique pattern of sines and cosines. Low dimensions oscillate fast (capturing nearby positions), high dimensions oscillate slowly (capturing distant structure).

This encoding is added to the word embedding, so the model sees both what a word means and where it sits. The sinusoidal design lets the model generalize to sequences longer than it was trained on.

The Transformer didn't just improve translation scores. It fundamentally changed what's computationally possible.

Property RNN/LSTM Before Transformer After
Processing order Sequential (word by word) Parallel (all words at once)
Long-range memory Degrades over distance (vanishing gradient) Direct connection between any two tokens
GPU utilization Poor (sequential bottleneck) Excellent (matrix multiplications parallelize)
Training time (WMT en-de) Days to weeks 12 hours (base) / 3.5 days (big)
Hardware 8+ GPUs, still slow 8 P100 GPUs, outperformed SOTA
Context handling Fixed hidden state, compresses everything Attention over full context window
Scalability Diminishing returns with size Scaling laws: more params = better (GPT, etc.)
Generality Good at sequences Text, images, audio, video, code, robotics...
Complexity per layer O(n × d²) sequential O(n² × d) parallel — quadratic in n, but parallelizable
Key tradeoff Linear in sequence length, but can't parallelize Quadratic in sequence length, but fully parallel

The Insight That Changed Everything

The bet was simple: sacrifice sequential efficiency for parallelism. RNNs process tokens one at a time in O(n) steps but can't use GPU parallelism. Transformers compare every token to every other token (O(n²)) but can do it all in one parallel operation.

For modern GPU hardware, O(n²) parallel beats O(n) sequential. This is why the transformer won — not because it's theoretically more efficient, but because it matches the hardware we actually have.

This one architectural choice is what enabled GPT, BERT, Claude, Gemini, DALL-E, Stable Diffusion, and everything else in the current AI wave.
