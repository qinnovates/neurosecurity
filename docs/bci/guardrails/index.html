BCI Security Guardrails

Working Draft (Concept Design) 2026-02-18

BCI security guardrails are not arbitrary policy decisions. They are derivable from physics. The equations governing electromagnetism, thermodynamics, and information theory already define what a BCI can and cannot do. Those same equations define what an attacker can and cannot do.

This document synthesizes a layered guardrail architecture starting from what physics gives us for free (signal integrity monitoring) and building toward what technology will eventually enable (on-chip encryption, anomaly detection, post-quantum protocols).

Architecture

Layer 0: Physics Boundary       [EXISTS]
Layer 1: Signal Integrity       [FEASIBLE TODAY]
Layer 2: Anomaly Detection      [NEAR-TERM]
Layer 3: Cryptographic Defense   [FUTURE]

Four Guardrail Layers

Each layer builds on the previous one. Layer 0 defines the boundary. Layers 1-3 are the actual security controls that use that boundary information to detect and respond to violations.

L0

Physics Boundary

EXISTS

The constraint system from the BCI Limits Equation. Defines the operating envelope: what is physically normal vs. abnormal for a given implant.

Every BCI has a physically defined operating envelope based on implant location, electrode count, power budget, and tissue properties. Anything outside this envelope is abnormal by definition.

L1

Signal Integrity Monitor

FEASIBLE TODAY

Lightweight, always-on signal integrity check running on the implant. Proven on NeuroPace RNS hardware (FDA-approved, 12+ year battery life).

Uses line length, signal energy, kurtosis, 1/f spectral exponent, and Higuchi fractal dimension. All O(n) or better. Maps directly to QIF coherence metric (Cs). When Cs drops below Cs_min, the guardrail triggers.

L2

Adaptive Anomaly Detection

NEAR-TERM

Per-patient baseline model that learns normal neural signatures and flags deviations. Catches subtle, targeted attacks that Layer 1 misses.

Detects slow-drift attacks, targeted perturbations tuned to patient signal characteristics, and replay attacks. Requires on-chip ML (~10-50 KB model). Update mechanism via Runemate Staves bytecode (sandboxed, no firmware reflash needed).

L3

Cryptographic Defense

FUTURE

Full NSP protocol on the implant: post-quantum key exchange, authenticated encryption, device authentication, and rollback-protected firmware updates.

AES-256-GCM-SIV symmetric encryption is viable on current hardware. The expensive part is the PQ handshake (ML-KEM + SPHINCS+), which only happens once per session. T1/T2 devices can run full NSP today. T3 implants need next-gen chips.

Signal Integrity Metrics

Layer 1 uses universal statistical properties of neural signals. These metrics work regardless of brain region because all cortical areas exhibit the same fundamental signal structure.

Metric What It Measures Compute Cost On-Chip Precedent
Line length Sum of absolute differences between consecutive samples O(n), trivial NeuroPace RNS (FDA-approved)
Signal energy Integral of absolute amplitude O(n), trivial NeuroPace RNS
1/f spectral exponent Slope of log-log power spectrum (scale-invariant) Short FFT, moderate FOOOF (validated in neuroscience)
Higuchi fractal dimension Signal complexity measure O(n * k_max), lightweight Validated for ECoG spike detection
Statistical moments Variance, kurtosis of amplitude distribution O(n), trivial Standard DSP

Connection to Cs: The coherence metric's four factors (phase variance, temporal entropy, gain fluctuation, spectral drift) map directly to these signal features. Cs_min becomes the threshold. When the composite score drops below Cs_min for a given brain function, the guardrail triggers.
