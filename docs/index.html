The Gap

"Unaltered" is not the same as "self-originating"

Five neurorights have been proposed to protect your brain: Mental Privacy, Cognitive Liberty, Mental Integrity, Psychological Continuity, and Equal Access. Philosophical frameworks define what these rights mean. None specify how a BCI system would actually enforce them.

The critical gap is in Mental Integrity. Current formulations protect against modification of neural signals — tampering, injection, disruption. But we are building systems that can inject, simulate, or reshape neural signals. A perfectly intact signal that was never generated by your brain passes every integrity check and violates everything MI is supposed to protect. Governance models must distinguish between "unaltered" and "self-originating."

That distinction requires engineering, not philosophy. It requires a metric that can verify whether a signal's statistical properties match your baseline neural signature. No framework had built that. So we did.

The Discovery

No one had catalogued the threats

FDA

Requires medical device cybersecurity assessments, but has no BCI-specific attack taxonomy. Guidance says "assess risk per ISO 14971" without defining what BCI threats look like.

MITRE

Published a medical device threat modeling playbook (2021), but it is methodology-generic. No neural-specific techniques. No BCI entries in ATT&CK.

CISA

Created a tabletop foresight game for BCI futures planning. A board game for brainstorming, not a security framework.

BCI Manufacturers

Neuralink, Synchron, Blackrock, Precision Neuroscience: zero public security documentation. No threat models, no vulnerability disclosure policies.

So I built it

I catalogued every known way to attack a brain-computer interface, scored each one for neural impact, traced the affected brain regions to psychiatric outcomes, and mapped the neurorights each technique could violate. 109 techniques. 61 diagnostic outcomes. 5 neurorights operationalized.

The Test

Does the framework hold up?

A framework that can't be tested is a story. Here's how the atlas validates against real hardware, real physics, and real neuroscience.

Hardware Validation

28 consumer sensor exploitation techniques validated with real devices. ANC earbuds extract ear canal biometrics at −47 dB below music. No firmware exploit. Just physics.

Endogenous Case Study

The framework's first test case: COVID + PTSD as a dual-vector endogenous attack chain. Same neural pathways, same failure modes, no device required.

Dual-Use Discovery

78 of 109 attack techniques have direct therapeutic counterparts. The same mechanism that treats Parkinson's can, if misconfigured, cause psychiatric harm.

Key Finding

The Neural Impact Chain

A six-stage research mapping that traces BCI attack techniques from their physical mechanism through the affected neural structure to their potential psychiatric outcome. 61 techniques map to symptoms matching DSM-5-TR criteria — from acute stress disorder to neurocognitive impairment. These mappings require clinical validation.

Applied Research

Project Runemate

A bytecode compiler that compresses BCI data 65-90%, making post-quantum encryption viable on low-power implantable devices. Protecting patients from unauthorized neural access requires encryption that fits within the power constraints of a brain implant. Runemate demonstrates this is feasible. Written in Rust. 200 KB on-chip.

The Origin

How a security engineer found a neuroscience problem

In 2021, COVID collided with PTSD from a traumatic event years earlier. My cognition fragmented in specific, measurable ways: memory consolidation disrupted, threat detection hyperactivated, self-model corrupted. As a security engineer, I recognized the pattern. These were the same failure modes I'd expect from a targeted attack on a brain-computer interface — except no device was involved.

The "attacker" was endogenous: inflammation and neuroplasticity changes, operating on the same neural substrate that BCI devices read and write. If biology can produce these failure modes naturally, what happens when we connect programmable devices to the same pathways?

Why Now

Before mass adoption, not after

Neuralink has implanted chips in human patients. Synchron's Stentrode is in FDA trials. Consumer EEG headsets sit on store shelves. These devices are tested for safety and efficacy — but no public framework exists to audit what they can do to a mind.

This dual-use reality means BCI security is not an engineering problem. It is a patient rights problem. Every proposed neurorights framework — cognitive liberty, mental privacy, psychological continuity — can be violated through technical pathways that already exist. The question is whether we build governance before the harm, or after.

IEEE's own BMI standards roadmap identifies BCI-specific security standards as an open standardization gap. This atlas is the first attempt to fill it. Open source. Open data. Ready for peer review.
